--------------------------------------------------------------------------------
1. PRODUCT SUMMARY
--------------------------------------------------------------------------------
LafdaMap: real-time, community-driven incident map (urban India, viral /
hyper-local). Users browse incidents on a map, report anonymously, and (later)
go live / watch streams per incident.

Locked scope decisions:
  - Geography: single-city MVP; default map center via env (e.g. Pune).
  - Incidents: no auto-expiry; moderation via Supabase Studio for now.

Out of scope for MVP (per PRD): named accounts, push notifications, upvotes,
SFU streaming, incident expiry automation, native apps.

--------------------------------------------------------------------------------
2. TECH STACK (AS IMPLEMENTED)
--------------------------------------------------------------------------------
  - Frontend: Next.js 14 (App Router), React 18, TypeScript, Tailwind CSS
  - UI: shadcn-style components (Button, tokens in app/globals.css)
  - Map: MapLibre GL via react-map-gl/maplibre + OpenFreeMap tiles (no API key)
  - Backend: Supabase (Postgres, Auth, Realtime)
  - Auth: Supabase anonymous sign-in (signInAnonymously), @supabase/ssr
  - Live streaming (Phase 5): WebRTC via simple-peer + Supabase signaling

Environment variables (see .env.example):
  - NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY
  - NEXT_PUBLIC_MAP_STYLE_URL (optional; default OpenFreeMap dark)
  - NEXT_PUBLIC_MAP_DEFAULT_CENTER (lat,lng,zoom)
