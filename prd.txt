1. Product Overview
1.1 Vision
LafdaMap is a real-time, community-driven incident mapping platform — think Google Maps but for street-level chaos. Users can browse, report, and go live on incidents happening near them, across categories like street fights, road rage, JCB excavations, and general chaos. The product is built for fun, virality, and hyper-local engagement, primarily targeting urban India.
1.2 Problem Statement
There is no lightweight, real-time way to see what is actually happening on the streets around you. Social media is noisy and non-spatial. Police helplines are reactive. WhatsApp groups are fragmented. LafdaMap fills this gap with an anonymous, map-first, real-time platform for citizen reporting and witnessing.

3.3 WebRTC Signaling Flow
LafdaMap uses Supabase as a lightweight WebRTC signaling server. No media server (TURN/SFU) is required for MVP since streams are peer-to-peer.
	•	Host taps 'Go Live' on an incident pin
	•	simple-peer creates an SDP offer and stores it in live_sessions via Supabase
	•	Viewers subscribe to the live_sessions row via Supabase Realtime
	•	Viewer's browser creates an SDP answer and sends it back to host
	•	ICE candidates are exchanged via a separate signaling channel (Supabase channel)
	•	P2P stream established — video + audio direct between peers
Limitation: P2P works well for 1-10 concurrent viewers. Scale beyond that requires a media relay (e.g. Cloudflare Calls or Mux) — post-MVP roadmap item.

4. Feature Requirements
4.1 Core Features — MVP
F1: Incident Map
	•	Dark-themed Mapbox map as the primary screen
	•	Show all active incidents as category-specific pins
	•	Live incidents show a pulsing ring animation around the pin
	•	Filter chips at top: All | Street Fight | Road Rage | JCB Khudai | Chaos
	•	User's current location shown as a distinct blue dot
	•	Tapping a pin opens the Incident Detail Card

F2: Incident Detail Card
	•	Slides up from bottom on pin tap
	•	Shows: category badge, title, description, time ago, reporter handle
	•	If live: shows viewer count and 'Watch Live' CTA
	•	If not live: shows 'Go Live Here' CTA
	•	Share button generates a deep link URL for the incident

F3: Report Incident (2-step flow)
	•	Step 1 — Category: 2x2 grid with emoji cards, tap to select
	•	Step 2 — Details: title input + GPS auto-detect location button
	•	Guest users can report without signing up
	•	On submit: incident inserted into Supabase and pin appears on all maps in real time

F4: Live Streaming (WebRTC)
	•	Host flow: tap 'Go Live' on any incident pin to start broadcasting
	•	Viewer flow: tap 'Watch Live' on a live incident pin
	•	In-stream HUD: LIVE badge, viewer count, category label
	•	Live chat: real-time text chat attached to the stream (Supabase Realtime channel)
	•	Host can end stream; is_active set to false, pin stops pulsing

F5: Guest Authentication
	•	On first open: supabase.auth.signInAnonymously() — no form, no email
	•	Guest session persisted in localStorage via Supabase client
	•	Reporter handle auto-generated: 'anonymous_[4-char-id]'
	•	No profile, no settings, no login screen — completely frictionless

5. User Experience Flows
5.1 Browse Flow
	•	User opens LafdaMap URL
	•	Anonymous session silently created in background
	•	Map loads, centered on user GPS location
	•	Nearby incident pins rendered from Supabase query
	•	User taps pin, sees detail card, can share or watch live
5.2 Report Flow
	•	User taps 'Report Incident' button
	•	Bottom sheet slides up — Step 1: select category
	•	Step 2: enter title + tap 'Detect Location'
	•	GPS coordinates captured via navigator.geolocation
	•	On confirm: POST to Supabase, pin appears on map in real time for all viewers
5.3 Go Live Flow
	•	User taps pin of an existing incident (or reports new one)
	•	Taps 'Go Live Here' in detail card
	•	Browser requests camera + mic permissions
	•	simple-peer creates offer, stored in live_sessions
	•	Live screen shown with LIVE badge, viewer counter, and chat
	•	Other users see pulsing pin and 'Watch Live' option
